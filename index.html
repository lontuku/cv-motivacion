Instagram Scraper Github
**Debido a que a Instagram no le parece ser scrapeado, no está disponible el scraper, solo el código del catálogo Rails, pero si les interesa revisarlo no tengo problemas en compartirlo.

Como el proyecto final para Desafío Latam, decidí hacer un catálogo de tiendas en Instagram de emprendedores chilenos. Para automatizar este proceso desarrollé un scraper de Instagram.

Características:
  • Descargar una cantidad ilimitada de fotos de cada perfil.
  • Obtener la cantidad de likes de cada imágen o video.
  • Obtener la cantidad de seguidores y seguidos de cada perfil.
  • Realizar un backup usando archivos .yml de la información recopilada.
  • Revisar perfiles ya scrapeados y determinar si ha subido nuevas imágenes o videos y realizar un update en la base de datos del catálogo rails, usando una API.
Tecnologías:
  • Selenium (Para automatizar el navegador)
  • Nokogiri (Facilitar el trabajo con los datos)
  • Regex (Para filtrar la información obtenida)
  • Xpath (Capturar la información relevante)
  • Ruby (Para organizar todo el proceso)
  • Rails (Para generar un catálogo online)
Desafíos:

Hubieron múltiples desafíos en este proyecto, la causa principal de dolores de cabeza es la forma en que IG estructura su sitio para complicar la extracción de información:

  • La exigencia de usar una cuenta válida. Tuve que crear una cuenta y cargar las cookies de la misma en cada solicitud para poder acceder a la información.
  • Como el uso de clases de css dinámicas (actualización reciente). Para lo que Xpath y regex fueron de increíble utilidad.
  • El que para optimizar la carga de imágenes está constantemente eliminando imágenes y cargando nuevas de forma dinámica.

Una vez recopilada y filtrada la información, la información fue tomada por una aplicación Rails que generaba un catálogo básico que ordena la información en las categorías correspondientes.

Scraper de Subtítulos
Demo Heroku Github

Una pequeña y práctica aplicación en ROR que permite scrapear la información de Subdivx (La base de datos de subtítulos en Español de mejor calidad).

Características:
  • Permite encontrar todos los subtítulos disponibles para cierto título ya sea serie o película.
  • Ordena los resultados obtenidos según que tan cercanos con en términos de palabras claves al título original.
  • Usa un drag&drop (con vanilla js), para capturar el nombre y realizar un ajax al controlador correspondiente en RAILS para luego generar una tabla ordenada por prioridad.
Tecnologías:
  • Rails (Generar una interfaz para la app)
  • Regex (Para poder filtrar la información tanto del título como los tags)
  • Nokogiri (Para facilitar el parse del sitio a scrapear)
Desafíos:
  • Mas que nada mi desconocimiento de Regex, para este proyecto aprendí Regex con distintos recursos online, este mismo conocimiento me facilitó bastante la realización del scraper de Instagram.
  • También fue primera vez que intenté hacer algo práctico con web-scraping por lo que también requirió adquirir conocimientos como el uso de Nokogiri y el uso de PRY para poder trabajar de forma mas cómoda.

Por otro lado sigue siendo mi proyecto mas práctico tanto como yo, como amigos/as lo usamos constantemente para descargar los subtítulos de series y películas.

Javascript

JS Turtle
Demo Github

Me llama mucho la atención el desarrollo de visualizaciones usando código, por lo mismo, cuando empecé a aprender js y las facilidades que tiene para usar el navegador como interfaz, trabajé en un generador de formas simples usando un turtle.

Características:
  • Los colores se distribuyen según si la iteración es o no un número primo.
  • Permite generar una enorme cantidad de figuras atractivas visualmente.
  • Permite compartir los diseños de múltiples formas:
    • Descargándolos como jpg.
    • Copiando los parámetros de generación en el portapaleles usando params.
  • Permite la generación aleatoria tanto de formas, como de las distintas capas de colores que las conforman.
Tecnologías:
  • Vanilla Javascript.
  • Un micro framework que dispone de una clase para utilizar un turtle en canvas.
Desafíos:
  • Generar formas visualmente atractivas, tomó algo de experimentación, usé Scratch para poder iterar mas rápido.
  • El agregar algunas de las funciones como el poder compartir los diseños o el poder copiar los parámetros en el portapeles de forma simple.
  • El poder generar una interfaz práctica que personas externas pudieran usar sin complicaciones.
